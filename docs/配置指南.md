# 配置指南（环境变量 / 同步流程 / 标签与业务视图）

本文用于指导在本项目中完成 **OpManager 对接、数据同步、以及 Dashboard 大屏正常展示** 所需的配置与约定。请按章节顺序阅读，并按「最小可用配置」清单逐项勾选。

---

## 1. 环境变量配置（`.env` / `.env.local`）

项目服务端会在启动时校验关键环境变量（见 `src/lib/env.ts`），缺失会导致 API/同步失败或页面空白。

### 1.1 必需变量

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `DATABASE_URL` | PostgreSQL 连接串 | `postgresql://user:pass@localhost:5432/monitoring-dashboard?schema=public` |
| `OPMANAGER_BASE_URL` | OpManager 地址（建议不要带末尾 `/`，代码已做兼容） | `https://ithelp.whrank.com:44443` |
| `OPMANAGER_API_KEY` | OpManager API Key | 从 OpManager 管理界面获取 |

### 1.2 可选但常用变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `OPMANAGER_TIMEOUT` | OpManager API 请求超时（毫秒） | `60000` |
| `OPMANAGER_USE_DEVICE_NAME_SUFFIX` | 见下节「企业版接口 name 后缀」 | 不设置或 `false` |
| `REDIS_URL` | Redis 连接串（若使用 Redis 缓存/广播） | 可选 |
| `COLLECT_METRICS_INTERVAL` | 指标采集间隔（秒） | `60` |
| `COLLECT_ALARMS_INTERVAL` | 告警采集间隔（秒） | `30` |
| `SYNC_TOPOLOGY_INTERVAL` | 业务视图/拓扑同步间隔（秒） | `300` |
| `SYNC_DEVICES_INTERVAL` | 设备同步间隔（秒） | `600` |
| `USE_MOCK_DATA` | 是否启用 Mock 数据 | 不设置或 `false` |
| `LOG_LEVEL` | 日志级别 | `info` |

### 1.3 企业版 / 非企业版差异：接口 name 后缀开关

部分**企业版** OpManager 的 `getInterfaces` 等接口，要求 `name` 参数必须带后缀 **`.10000000001`**；**非企业版**通常直接使用 `listDevices.deviceName`（常见为 IP/hostname）即可。

通过环境变量控制（见 `src/services/opmanager/client.ts`）：

- **`OPMANAGER_USE_DEVICE_NAME_SUFFIX`**
  - `true`：企业版，会额外尝试 `name + ".10000000001"`
  - `false` 或不设置：非企业版，不追加后缀

示例：

```env
# 非企业版（默认）
# OPMANAGER_USE_DEVICE_NAME_SUFFIX 不设置即可

# 企业版（需要后缀）
OPMANAGER_USE_DEVICE_NAME_SUFFIX="true"
```

若接口同步一直为 0，优先检查此项，并用 OpManager 接口验证：  
`GET {OPMANAGER_BASE_URL}/api/json/device/getInterfaces?name=<deviceName>` 或 `name=<deviceName>.10000000001`。

## 2. 首次初始化与同步顺序（推荐）

为了让 Dashboard 不空白，建议按以下顺序执行：

1. **设备同步**
   - 入口：**`/admin/devices`** → 点击「同步设备」
   - 作用：将 OpManager 的设备写入 DB，并写入 `type` / `category` / `status` / `isMonitored`

2. **接口同步**
   - 入口：**`/admin/interfaces`** → 点击「同步接口」
   - 作用：将接口写入 DB（`Interface` 表）；依赖已同步的设备

3. **（可选）告警/指标采集**
   - 视部署方式而定（采集器/定时任务），用于让图表与告警滚动更丰富
   - 采集周期可通过环境变量配置（见 1.2）：`COLLECT_METRICS_INTERVAL`、`COLLECT_ALARMS_INTERVAL`、`SYNC_TOPOLOGY_INTERVAL`

> 若接口同步后仍为 0，一般是 `getInterfaces` 的 `name` 形态不匹配，先检查 `OPMANAGER_USE_DEVICE_NAME_SUFFIX`。

## 3. Dashboard 展示规则：不设置 vs 设置（重要）

项目中 **“设置”** 指：选择某个**标签（tag）**或**业务视图（Business View）**来过滤展示范围。

### 3.1 行为总结（一句话）

- **不设置（未选标签/未选视图）**：展示**全量/全局**数据，保证首次部署就有内容。
- **有设置（选了标签或视图）**：**优先按设置过滤**；若没有任何设备/接口带有该标签或属于该视图，则对应面板会显示「无匹配数据」或提示；部分组件会**兜底展示全量**并提示「请在管理后台添加标签」。

### 3.2 不设置时（默认）

- 使用全量接口，例如：
  - `GET /api/dashboard/overview`（从 DB 统计设备/告警）
  - `GET /api/devices?limit=...`（全量设备）
  - `GET /api/alarms?limit=...`（全量告警滚动）
  - 接口流量：`GET /api/interfaces/realtime-traffic?tag=__all__`（全量 Top 流量）

### 3.3 设置时（按标签/业务视图）

- 会按所选标签或业务视图严格过滤。
- **标签/视图不会自动产生数据**，只是过滤条件；要让「设置后也有数据」，必须在管理后台把该**标签打到设备或接口**上，或确保**业务视图在 OpManager 中已包含设备**且在本地已配置同名视图。

## 4. 大屏各面板与默认标签/数据源对应关系

大屏路由：**`/dashboard`**（或 ` /app/dashboard`）。三个 Tab：**全屏总览**、**硬件服务器**、**网络设备**。

### 4.1 Tab「全屏总览」

| 面板名称 | 数据来源 / 默认约定 | 说明 |
|----------|----------------------|------|
| 设备总览环形图 | `GET /api/dashboard/overview`（`devices.totalAll`、`byType`） | 全量统计，不依赖标签 |
| **核心资产状态** | `TaggedDevicePanel`，默认 **`tag="重要设备"`** | 仅展示带有 **`重要设备`** 标签的设备；若未给任何设备打该标签，组件会兜底请求全量设备以避免左侧空白 |
| 拓扑图 | 拓扑数据（本地/OpManager） | 与标签无关 |
| 全网实时告警 | `GET /api/alarms?limit=20` | 全量告警 |

### 4.2 Tab「硬件服务器」

| 面板名称 | 数据来源 / 默认约定 | 说明 |
|----------|----------------------|------|
| 标签筛选下拉 | `GET /api/devices/tags` | 可选「全部服务器」或某设备标签 |
| 业务视图服务器分组 / 标签列表 | 未选标签：`GET /api/dashboard/grouped-devices`（按业务视图）；选标签：`GET /api/dashboard/tagged-devices?tag=...&deviceType=servers` | 有设置则按标签过滤 |
| 服务器集群性能趋势 | 设备指标（可带标签过滤） | 同上 |

### 4.3 Tab「网络设备」

| 面板名称 | 数据来源 / 默认约定 | 说明 |
|----------|----------------------|------|
| 标签筛选下拉 | `GET /api/devices/tags` | 可选「全部网络设备」或某设备标签 |
| 业务视图分组 / 标签列表 | 同「硬件服务器」，`deviceType=network` | 有设置则按标签过滤 |
| 核心网络设备性能趋势 | 设备指标，默认 8 台 | 与标签可选关联 |
| **上联出口流量监控** | `TaggedInterfaceTrafficWidget`，**默认 `defaultTag="上联"`** | 见下节 |
| **互联网出口流量监控** | `TaggedInterfaceTrafficWidget`，**默认 `defaultTag="互联网出口"`** | 见下节 |

### 4.4 两个「接口流量」面板的约定标签（必读）

- **上联出口流量监控**：默认按接口标签 **`上联`** 筛选；若没有任何接口带有标签 `上联`，会显示提示「未找到标签『上联』的接口…」，并**兜底展示全量接口 Top 流量**。
- **互联网出口流量监控**：默认按接口标签 **`互联网出口`** 筛选；逻辑同上，兜底为全量 Top 流量。

要让这两个面板「按标签有数据」：请到 **`/admin/interfaces`** 给对应接口打上 **`上联`** 或 **`互联网出口`** 标签（见第 5 节）。

## 5. 标签（Tags）配置指南（强烈建议统一约定）

标签为字符串数组，**大小写与空格敏感**。建议全项目统一写法，避免「重要设备」/「重要设备 」/「IMPORTANT」混用导致筛选不到。

### 5.1 设备标签（Device tags）

- **入口**：**`/admin/devices`**
- **用途**：大屏「核心资产状态」、服务器/网络设备 Tab 的「按标签筛选」、业务视图分组旁的标签模式等。
- **建议标签（示例）**：
  - **`重要设备`**：总览页「核心资产状态」默认使用，建议给核心设备打上。
  - `核心设备`、`核心服务器`、`交换机`、`路由器`、`防火墙`、`出口`、`汇聚`、`接入` 等按需使用。

**操作步骤**：在设备列表中找到目标设备 → 点击该行的「编辑」→ 在标签输入框中填写，多个标签用英文逗号分隔（如 `重要设备, 核心服务器`）→ 保存。

### 5.2 接口标签（Interface tags）— 重点：上联 / 互联网出口

- **入口**：**`/admin/interfaces`**
- **用途**：网络设备 Tab 下的「上联出口流量监控」「互联网出口流量监控」等按接口标签筛选。

**建议约定标签（与大屏默认一致）**：

- **`上联`**：用于上联出口流量面板。
- **`互联网出口`**：用于互联网出口流量面板。
- 其他可按需：`关键链路`、`出口链路`、`核心上联` 等（若大屏或 API 使用相同名称即可被筛选）。

**为接口打标签的步骤**：

1. 打开 **`/admin/interfaces`**。
2. 可用顶部「按标签筛选」或搜索框定位到目标接口（如某台交换机的上联口、互联网出口口）。
3. 在该接口行的「标签 (自定义)」列点击 **「编辑」**。
4. 在输入框中填写标签，多个用英文逗号分隔，例如：`上联` 或 `互联网出口` 或 `上联, 关键链路`。
5. 点击 **「保存」**。
6. 确认：在「按标签筛选」中选择「上联」或「互联网出口」，应能看到刚打的接口；或直接刷新大屏「网络设备」Tab，对应流量面板应出现按标签的数据（否则会显示兜底全量并提示）。

## 6. 业务视图（Business View）配置指南

大屏「硬件服务器」「网络设备」Tab 中「按业务视图分组」的面板，依赖：

1. **OpManager 侧**：在 OpManager 中已创建业务视图（Business View），并把设备加入该视图。
2. **本系统**：表 `BusinessViewConfig` 中有一条**名称与 OpManager 完全一致**的视图配置（仅名称映射，设备归属以 OpManager 为准）。

### 6.1 在 OpManager 中（前置条件）

- 创建业务视图，并给视图起一个**内部名称**（如「出口业务」）。
- 将需要分组的设备加入该业务视图。

### 6.2 在本系统中维护业务视图名称

- **入口 1**：**`/admin/topology`**（推荐）  
  - 点击「Add New View」：  
    - **OpManager View Name**：填写与 OpManager 中**完全一致**的视图名称（如 `出口业务`）。  
    - **Display Name**：展示用名称（如「出口业务」）。  
  - 保存后，该视图会出现在「按业务视图分组」的数据源中。

- **入口 2**：脚本初始化  
  - 运行 `scripts/init-business-views.ts` 可写入默认业务视图（如「核心业务视图」）；若 OpManager 中已有同名视图，大屏即可按该视图分组。

### 6.3 数据流说明

- **分组接口**：`GET /api/dashboard/grouped-devices` 会读取 `BusinessViewConfig` 中所有 `isActive: true` 的视图，对每个视图调用 OpManager 的 `getBusinessDetailsView(view.name)` 获取该视图下的设备，再与本地 DB 设备做类型/状态等匹配后返回分组结果。
- 因此：**不需要在本系统给设备「绑定」业务视图**，只需在 **OpManager 中把设备加入视图**，并在本系统 **`/admin/topology` 中维护同名视图** 即可。

### 6.4 若未配置业务视图

- `GET /api/dashboard/grouped-devices` 会返回 `groups: []`。
- 对应面板会显示「暂无业务视图数据」；此时仍可通过「标签筛选」使用按设备标签的列表。

## 7. 主要 API 速查（与 Dashboard / 管理后台相关）

| 接口 | 说明 |
|------|------|
| `GET /api/dashboard/overview` | 总览统计（设备总数、按类型、告警等） |
| `GET /api/devices` | 设备列表（支持分页、标签等） |
| `GET /api/devices/tags` | 设备已有标签列表 |
| `PUT /api/devices/:id/tags` | 更新设备标签 |
| `GET /api/interfaces` | 接口列表（分页、按标签/搜索） |
| `GET /api/interfaces/tags` | 接口已有标签列表 |
| `PUT /api/interfaces/:id/tags` | 更新接口标签 |
| `GET /api/interfaces/realtime-traffic?tag=__all__` | 全量接口 Top 流量 |
| `GET /api/interfaces/tagged-traffic?tag=<tag>` | 按接口标签的流量 |
| `GET /api/dashboard/grouped-devices` | 按业务视图分组的设备 |
| `GET /api/dashboard/tagged-devices?tag=...&deviceType=...` | 按设备标签+类型筛选 |
| `GET /api/alarms?limit=...` | 告警列表（大屏滚动） |
| `GET /api/admin/views` | 业务视图配置列表（管理用） |
| `POST /api/admin/views` | 新增业务视图配置 |

## 8. 常见问题排查（Dashboard 仍然没数据）

### 8.1 `/dashboard` 总览数字为 0

1. 浏览器直接访问 `GET /api/dashboard/overview` 看返回。
2. 若 `devices.totalAll` 等为 0，说明 DB 内设备/告警未写入 → 回到 **第 2 节** 做设备同步（及可选告警采集）。

### 8.2 「核心资产状态」为空或很少

- 该面板默认只显示带 **`重要设备`** 标签的设备。
- 处理：到 **`/admin/devices`** 给需要展示的设备打上 **`重要设备`** 标签（若不想用标签，可改组件不传 `tag`，则显示全量）。

### 8.3 「上联出口」「互联网出口」流量面板为空或提示需接口标签

- 原因：没有给任何接口打 **`上联`** 或 **`互联网出口`** 标签。
- 处理：到 **`/admin/interfaces`** 给对应上联口/互联网出口口打上 **`上联`**、**`互联网出口`**（见 **5.2**）。
- 说明：当前逻辑会兜底展示全量 Top 流量并提示「请在 /admin/interfaces 为接口添加该标签」。

### 8.4 接口已同步，但「关键接口流量」类面板仍无按标签数据

- 确认接口是否已打标签，且标签名与面板使用的**完全一致**（含空格、大小写）。
- 确认 `GET /api/interfaces/tags` 能返回你设置的标签；再试 `GET /api/interfaces/tagged-traffic?tag=<你的标签>`。

### 8.5 同步接口一直为 0

- 重点检查 **`OPMANAGER_USE_DEVICE_NAME_SUFFIX`**（企业版设为 `true`）。
- 用 OpManager 原生接口验证：  
  `GET {OPMANAGER_BASE_URL}/api/json/device/getInterfaces?name=<deviceName>` 或带后缀再试。

### 8.6 业务视图分组为空

- 确认 OpManager 中已创建业务视图且已加入设备。
- 确认在 **`/admin/topology`** 已添加**同名**视图（OpManager View Name 与 OpManager 完全一致）。
- 确认该视图在 DB 中 `isActive: true`（默认创建即为 true）。

## 9. 推荐的「最小可用配置」清单（Checklist）

- [ ] `.env` / `.env.local` 配置：`DATABASE_URL`、`OPMANAGER_BASE_URL`、`OPMANAGER_API_KEY`
- [ ] 设备同步完成（`/admin/devices` → 同步设备，DB 有 `Device` 数据）
- [ ] 接口同步完成（`/admin/interfaces` → 同步接口，DB 有 `Interface` 数据）
- [ ] （可选）给至少若干台设备打上 **`重要设备`** 标签（总览「核心资产状态」）
- [ ] （可选）给上联口打 **`上联`**、互联网出口口打 **`互联网出口`**（网络设备 Tab 两个流量面板）
- [ ] （可选）在 OpManager 创建业务视图并加设备，在 **`/admin/topology`** 添加同名视图（分组面板）

按上述清单完成后，大屏应能显示总览数字、核心资产（或全量兜底）、告警、拓扑；网络设备 Tab 两个流量面板至少会有兜底全量或按标签数据。

## 10. 故障排查流程简图（文字版）

```
Dashboard 无数据 / 某面板为空
    │
    ├─ 总览数字为 0？
    │     → 查 /api/dashboard/overview
    │     → 若为 0：先做设备同步（/admin/devices）
    │
    ├─ 「核心资产状态」为空？
    │     → 给设备打「重要设备」标签（/admin/devices）
    │
    ├─ 「上联/互联网出口」无按标签数据？
    │     → 给对应接口打「上联」「互联网出口」（/admin/interfaces）
    │     → 仍无则看兜底是否展示全量；检查 /api/interfaces/tags
    │
    ├─ 接口同步一直为 0？
    │     → 检查 OPMANAGER_USE_DEVICE_NAME_SUFFIX（企业版 true）
    │     → 用 OpManager getInterfaces 接口验证 name 形态
    │
    └─ 业务视图分组为空？
          → OpManager 中视图已创建且含设备？
          → /admin/topology 中已添加同名视图？
```
