## 🔧 代码修复详情

### 问题根源

1. **映射优先级错误**
   - 原代码默认返回 `OFFLINE`
   - "Trouble" 状态被包含在 "error/critical/trouble" 检查中，但优先级低

2. **字段名称问题**
   - 代码检查 `device.status`
   - 但 OpManager 实际返回 `statusStr`
   - 导致映射失败

3. **isManaged 字段缺失**
   - OpManager 没有返回 `isManaged` 字段（undefined）
   - 原代码默认判断为 `false`（未监控）
   - 实际上能获取到数据说明在监控中

### 修复内容

#### 修复 1: 状态映射优先级

```typescript
// 修复前
return 'OFFLINE'; // 默认离线

// 修复后
if (statusStr.includes('critical')) return 'ERROR';
if (statusStr.includes('trouble')) return 'ERROR';  // ⚠️ 明确处理 Trouble
// ...
return 'ONLINE'; // 默认在线（能获取数据 = 设备可达）
```

#### 修复 2: 使用正确的字段

```typescript
// 修复前
status: mapDeviceStatus(device.status)

// 修复后
const actualStatus = device.statusStr || device.status || '';
status: mapDeviceStatus(actualStatus)
```

#### 修复 3: isManaged 智能判断

```typescript
// 修复前
isMonitored: String(device.isManaged).toLowerCase() === 'true'
// undefined → false

// 修复后
const isMonitored = opIsManagedRaw === undefined
  ? (actualStatus.toLowerCase() !== 'unmanaged')  // 根据状态判断
  : (String(opIsManagedRaw).toLowerCase() === 'true');
```
