## 🔍 状态映射分析

### OpManager 的判断

OpManager 将设备标记为 **"Trouble"（故障）** 是基于：

1. ✅ 设备本身在线（可以通过 SNMP 采集数据）
2. ⚠️ **但存在严重问题**：
   - 1 个 MAJOR 告警（接口流量 100%）
   - 9 个 MINOR 告警（接口故障）

### 我们系统的映射规则

```typescript
function mapDeviceStatus(opManagerStatus: string): string {
  const statusStr = (opManagerStatus || '').toLowerCase();
  
  if (statusStr.includes('trouble') || statusStr.includes('error') || statusStr.includes('critical'))
    return 'ERROR';  // ⚠️ "Trouble" 应该映射到这里
  
  if (statusStr.includes('warning') || statusStr.includes('warn'))
    return 'WARNING';
  
  if (statusStr.includes('up') || statusStr.includes('online') || statusStr.includes('clear'))
    return 'ONLINE';
  
  if (statusStr.includes('down') || statusStr.includes('offline') || statusStr.includes('unmanaged'))
    return 'OFFLINE';
  
  return 'OFFLINE'; // 默认
}
```

### 映射结果

```
OpManager: statusStr="Trouble"  →  应该映射为: ERROR 或 WARNING
```

## 💡 问题分析

### 为什么显示为 OFFLINE 而不是 ERROR？

**可能的原因**：

1. **`isManaged` 字段缺失**
   - OpManager API 没有返回 `isManaged` 字段
   - 代码默认判断为 `false`（未监控）
   - 因此被映射为 OFFLINE

2. **状态判断优先级问题**
   - 可能先判断了 `isManaged=false`
   - 直接映射为 OFFLINE
   - 没有检查实际的 `statusStr="Trouble"`
