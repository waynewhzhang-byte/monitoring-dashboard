# 监控大屏系统架构设计

## 技术栈

### 前端层
- **Next.js 14+** (App Router)
- **TypeScript**
- **TailwindCSS** + **Framer Motion** (动画)
- **Recharts/ECharts** (图表库)
- **React Flow** (网络拓扑图)
- **Zustand** (状态管理)
- **Socket.io-client** (实时通信)

### 后端层
- **Next.js API Routes** / **Server Actions**
- **Prisma ORM**
- **PostgreSQL** (持久化存储)
- **Redis** (缓存 + 实时数据)
- **Socket.io** (WebSocket服务)
- **Node-cron** (定时任务)

### 外部集成
- **ManageEngine OpManager REST API**
- **SNMP协议** (可选，用于直接设备采集)

## 数据流设计

### 1. 实时监控数据流

```
OpManager API → Collector (cron) → Redis (cache) 
                     ↓
              PostgreSQL (历史)
                     ↓
              Socket.io (推送) → 前端实时更新
```

### 2. 拓扑配置数据流

```
前端编辑器 → API Route → Prisma → PostgreSQL
                                      ↓
                              前端订阅更新
```

### 3. 告警数据流

```
OpManager Alarms → Collector → Redis (实时告警队列)
                                    ↓
                          告警规则引擎 (过滤/分级)
                                    ↓
                      ┌──────────┴──────────┐
                      ↓                     ↓
              Socket.io (推送)    PostgreSQL (存储)
                      ↓
              前端告警面板
```

## 性能优化策略

### 1. 数据采集优化
- **差量采集**: 仅获取变化的指标
- **批量请求**: 合并多个设备的API调用
- **智能轮询**: 根据设备重要性调整频率
- **缓存策略**: Redis缓存热点数据（TTL: 30s）

### 2. 前端渲染优化
- **虚拟滚动**: 大量告警列表使用react-window
- **Canvas渲染**: 复杂拓扑图使用Canvas而非SVG
- **数据抽样**: 长时间序列图表进行降采样
- **懒加载**: 非可视区域组件延迟加载

### 3. 实时通信优化
- **消息压缩**: gzip压缩WebSocket消息
- **批量推送**: 100ms内的更新合并推送
- **客户端节流**: 前端防抖处理高频更新
- **房间隔离**: 不同大屏订阅不同Socket.io房间

## 安全设计

### API安全
- **认证**: JWT Token + API Key
- **授权**: RBAC角色权限
- **限流**: Redis + rate-limiter-flexible
- **加密**: HTTPS + WebSocket Secure

### 数据安全
- **敏感信息**: 环境变量 + Vault
- **SQL注入**: Prisma参数化查询
- **XSS防护**: Next.js内置防护 + DOMPurify

## 监控和运维

### 系统监控
- **应用监控**: Sentry (错误追踪)
- **性能监控**: Next.js Analytics
- **日志**: Winston + 集中日志平台

### 告警通知
- **WebSocket断线**: 邮件/钉钉通知
- **数据采集失败**: 自动重试 + 告警
- **性能降级**: 自动切换降级策略
