## 🔍 原因分析

### 为什么所有设备都是 OFFLINE？

OpManager 返回的设备状态为 `UnManaged`（未管理），我们的系统将其映射为 `OFFLINE`：

```typescript
// OpManager 状态映射规则
function mapDeviceStatus(opManagerStatus: string): string {
  const statusStr = (opManagerStatus || '').toLowerCase();
  
  if (statusStr.includes('up') || statusStr.includes('online') || statusStr.includes('clear')) 
    return 'ONLINE';
  
  if (statusStr.includes('down') || statusStr.includes('offline') || statusStr.includes('unmanaged')) 
    return 'OFFLINE';
  
  // ...
  return 'OFFLINE';
}
```

**OpManager 中的 UnManaged 状态意味着**:
- 设备已添加到 OpManager
- 但未启用主动监控
- 不会采集性能数据
- 不会生成告警（除非特殊配置）

## 💡 解决方案

### 方案 1: 在 OpManager 中启用设备监控（推荐）

#### 步骤：

1. **登录 OpManager 管理界面**
   ```
   URL: https://ithelp.whrank.com:44443
   ```

2. **找到设备列表**
   - 导航到 `Inventory` → `Devices`
   - 或搜索设备名称/IP

3. **启用设备监控**
   - 选择设备
   - 右键或点击操作菜单
   - 选择 `Manage Device` 或 `Enable Monitoring`
   - 确认操作

4. **等待状态更新**
   - OpManager 会开始监控设备
   - 状态会从 `UnManaged` 变为 `Up`/`Down`/`Clear` 等

5. **重新同步到我们的系统**
   ```powershell
   # 手动触发设备同步
   curl http://localhost:3000/api/devices/sync -X POST
   ```

6. **验证结果**
   ```powershell
   npx tsx check-device-status.ts
   ```

## 📝 理解设备状态

### OpManager 中的设备状态

| OpManager 状态 | 含义 | 我们系统的映射 |
|---------------|------|--------------|
| `Clear` | 正常，无告警 | ONLINE |
| `Up` | 在线 | ONLINE |
| `Down` | 离线/宕机 | OFFLINE |
| `UnManaged` | 未管理/未监控 | OFFLINE |
| `Critical` | 严重故障 | ERROR |
| `Warning` | 警告 | WARNING |

### 监控状态 (isMonitored)

- **已监控** (`true`): OpManager 中 `isManaged = "true"`
  - 会采集性能数据
  - 会生成告警
  - 会显示实时状态
  
- **未监控** (`false`): OpManager 中 `isManaged = "false"` 或 `UnManaged`
  - 不采集性能数据
  - 不生成告警
  - 只作为记录存在

## 🔄 同步逻辑

我们的系统会忠实反映 OpManager 的状态：

```typescript
// 设备同步时的状态映射
const deviceData = {
  status: mapDeviceStatus(device.status),        // OFFLINE (因为 UnManaged)
  isMonitored: String(device.isManaged).toLowerCase() === 'true',  // false
  // ...
};
```

**这意味着**:
- 我们的系统显示的状态**准确反映** OpManager 的状态
- 如果 OpManager 中设备是 `UnManaged`，我们显示为 `OFFLINE`
- 这不是 bug，而是正确的数据映射
