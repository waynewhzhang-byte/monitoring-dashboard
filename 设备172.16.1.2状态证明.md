## 📊 状态分析

### OpManager 的判断

OpManager 将此设备标记为 **"Trouble"（故障）**，而不是：
- ❌ 不是 "Up"（正常）
- ❌ 不是 "Clear"（无问题）
- ❌ 不是 "Down"（完全离线）
- ✅ **是 "Trouble"（有问题）**

### 为什么 OpManager 认为设备有问题？

可能的原因（根据 OpManager 的判断逻辑）：

1. **有告警存在**
   - 设备可能有活动告警
   - 例如：接口故障、性能问题、配置问题

2. **性能指标异常**
   - CPU 使用率过高
   - 内存使用率过高
   - 接口错误率高

3. **部分功能故障**
   - 某些接口处于 Down 状态
   - SNMP 采集异常
   - 配置备份失败

4. **监控状态异常**
   - 设备响应缓慢
   - 数据包丢失率高

### 我们系统的映射

根据我们的映射规则：

```typescript
function mapDeviceStatus(opManagerStatus: string): string {
  const statusStr = (opManagerStatus || '').toLowerCase();
  
  if (statusStr.includes('up') || statusStr.includes('online') || statusStr.includes('clear')) 
    return 'ONLINE';
  
  if (statusStr.includes('down') || statusStr.includes('offline') || statusStr.includes('unmanaged')) 
    return 'OFFLINE';
  
  if (statusStr.includes('warning') || statusStr.includes('warn')) 
    return 'WARNING';
  
  if (statusStr.includes('error') || statusStr.includes('critical') || statusStr.includes('trouble')) 
    return 'ERROR';  // ⚠️ "Trouble" 会被映射到这里
  
  return 'OFFLINE'; // 默认
}
```

**"Trouble"** 状态应该被映射为 **ERROR**，而不是 OFFLINE。
