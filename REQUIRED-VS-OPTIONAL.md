# ✅ 必需配置 vs ⚠️ 可选配置

> 快速了解哪些配置是必需的，哪些是可选的

---

## 🎯 一图看懂

```
┌─────────────────────────────────────────────────────────────┐
│                 监控大屏配置优先级                            │
└─────────────────────────────────────────────────────────────┘

【必需配置】- 没有这些，系统无法正常工作
    │
    ├─ ✅ 环境变量 (.env)
    │       ├─ DATABASE_URL          → 数据库连接
    │       ├─ REDIS_URL             → 缓存连接
    │       ├─ OPMANAGER_BASE_URL    → OpManager 地址
    │       └─ OPMANAGER_API_KEY     → OpManager 密钥
    │
    ├─ ✅ Business View 配置 ⭐ 【核心】
    │       ├─ 在 /admin/topology 添加
    │       ├─ 名称必须与 OpManager 完全一致
    │       ├─ 必须激活 (isActive = true)
    │       └─ 用于拓扑显示和设备分组
    │
    ├─ ✅ 设备数据同步
    │       ├─ 在 /admin/devices 手动触发
    │       └─ 或调用 API: POST /api/devices/sync
    │
    └─ ✅ 采集器运行
            └─ npm run collector （持续运行）

【可选配置】- 增强功能，有了更好，没有也能用
    │
    ├─ ⚠️ 设备标签 (Device Tags)
    │       ├─ 用于分组和过滤
    │       ├─ 不影响基本显示
    │       └─ 大型网络环境推荐使用
    │
    ├─ ⚠️ 接口标签 (Interface Tags)
    │       ├─ 用于接口分类
    │       ├─ 不影响流量采集
    │       └─ 按需配置
    │
    ├─ ⚠️ 设备类型 (Device Type)
    │       ├─ 从 OpManager 自动获取
    │       ├─ 可手动调整
    │       └─ 用于统计和图标显示
    │
    └─ ⚠️ 自定义大屏配置
            ├─ 预设模板已足够
            ├─ 可自定义 Widget 布局
            └─ 高级用户可创建新配置
```

## 📊 详细对比表

| 配置项 | 类型 | 必需性 | 用途 | 不配置会怎样？ |
|-------|------|--------|------|---------------|
| **DATABASE_URL** | 环境变量 | ✅ 必需 | 数据库连接 | ❌ 系统无法启动 |
| **OPMANAGER_BASE_URL** | 环境变量 | ✅ 必需 | OpManager API | ❌ 无法采集数据 |
| **OPMANAGER_API_KEY** | 环境变量 | ✅ 必需 | API 认证 | ❌ API 调用失败 |
| **Business View** | 数据库配置 | ✅ 必需 | 拓扑显示 | ❌ 拓扑图为空，设备无法按业务分组 |
| **设备同步** | 操作 | ✅ 必需 | 获取设备数据 | ❌ 设备列表为空，大屏无数据 |
| **采集器运行** | 服务 | ✅ 必需 | 持续更新数据 | ❌ 数据不更新，显示过期数据 |
| **设备标签 (Tags)** | 数据库字段 | ⚠️ 可选 | 分组过滤 | ✅ 能正常显示，但无法按标签过滤 |
| **接口标签** | 数据库字段 | ⚠️ 可选 | 接口分类 | ✅ 流量数据正常，无法按标签过滤 |
| **设备类型** | 数据库字段 | ⚠️ 可选 | 分类统计 | ✅ 自动从 OpManager 获取 |
| **Redis** | 服务 | ⚠️ 可选 | 缓存加速 | ✅ 能正常运行，但性能稍差 |
| **自定义大屏** | 配置文件 | ⚠️ 可选 | 个性化展示 | ✅ 使用预设模板 |

## 🚀 快速启动最小配置

### 步骤 1: 环境配置（3 分钟）

```bash
# .env 文件（只需要这 4 项）
DATABASE_URL="postgresql://postgres:password@localhost:5432/monitoring_dashboard"
REDIS_URL="redis://localhost:6379"
OPMANAGER_BASE_URL="https://10.141.69.192:8061"
OPMANAGER_API_KEY="your-api-key"
```

### 步骤 2: Business View 配置（5 分钟）⭐

1. 启动应用：`npm run start`
2. 访问：`http://localhost:3000/admin/topology`
3. 添加 Business View：
   - OpManager View Name: `出口业务`（与 OpManager 一致）
   - 激活：✅
4. 保存

### 步骤 3: 同步设备（2 分钟）

1. 访问：`http://localhost:3000/admin/devices`
2. 点击：**"Sync Devices from OpManager"**
3. 等待完成

### 步骤 4: 启动采集器（1 分钟）

```bash
npm run collector
```

### 步骤 5: 验证（1 分钟）

```bash
npm run verify:data-flow
```

**期望结果**：通过率 ≥ 80%

### 步骤 6: 访问大屏（立即）

```
http://localhost:3000/dashboards
```

**总耗时**：约 12 分钟 ⏱️

## ❓ 常见疑问

### Q1: 必须添加标签吗？

**答案：不需要！**

- ❌ 标签不是必需的
- ✅ 系统可以在没有标签的情况下正常运行
- ✅ 所有核心功能都不依赖标签
- 💡 标签是可选的增强功能，用于分组和过滤

**何时需要标签？**
- 设备数量较多（50+ 台）
- 需要按业务分类
- 需要生成分类报表

### Q2: Business View 是什么？必须配置吗？

**答案：必须配置！**

Business View 是 OpManager 中的概念，用于：
- 定义网络拓扑结构
- 将设备按业务分组
- 关联设备和拓扑图

**在本系统中**：
- ✅ **必须配置** Business View 才能显示拓扑图
- ✅ Business View 连接 OpManager 和本系统
- ✅ 采集器会自动同步 Business View 中的设备和拓扑

**配置位置**：
- OpManager → 定义 Business View
- Admin 面板 → `/admin/topology` → 添加配置

### Q3: 设备同步是自动的吗？

**答案：部分自动**

| 数据类型 | 同步方式 | 说明 |
|---------|---------|------|
| 设备 (Device) | ⚠️ 手动 | 需要在 Admin 面板手动触发 |
| 接口 (Interface) | ⚠️ 手动 | 需要手动触发 |
| 指标 (Metric) | ✅ 自动 | 每 60 秒自动采集 |
| 告警 (Alarm) | ✅ 自动 | 每 30 秒自动采集 |
| 拓扑 (Topology) | ✅ 自动 | 每 5 分钟自动采集 |

**为什么设备和接口是手动的？**
- 设备和接口变化不频繁
- 手动同步可以避免频繁 API 调用
- 管理员可以控制同步时机

### Q4: 如果不配置 Redis 会怎样？

**答案：能正常运行，但性能稍差**

- ✅ 系统可以在没有 Redis 的情况下运行
- ✅ 数据会直接从数据库读取
- ⚠️ 性能可能稍差（特别是在大量请求时）
- 💡 小型部署（< 50 台设备）可以不用 Redis

### Q5: 预设大屏模板够用吗？需要自定义吗？

**答案：大部分情况下够用**

系统提供 4 个预设模板：
1. **网络监控大屏** - 设备状态、流量、拓扑
2. **告警中心大屏** - 告警统计、趋势分析
3. **服务器监控大屏** - 服务器性能监控
4. **综合监控大屏** - 全方位监控

**何时需要自定义？**
- 有特殊的展示需求
- 需要特定的 Widget 组合
- 需要调整布局和样式

4. ✅ 启动采集器

## 🎉 总结

### 记住这三点

1. **Business View 是核心** ⭐
   - 必须配置
   - 连接 OpManager 和大屏
   - 用于拓扑显示

2. **标签是可选的** 🏷️
   - 不是必需的
   - 用于增强分组和过滤
   - 按需添加

3. **最小配置即可启动** 🚀
   - 4 个环境变量
   - 1 个 Business View
   - 1 次设备同步
   - 1 个采集器进程

### 推荐流程

```
首次部署 → 只配置必需项 → 验证运行正常 → 逐步添加可选配置
```

**不要一开始就追求完美配置，先让系统跑起来！** 🎯
